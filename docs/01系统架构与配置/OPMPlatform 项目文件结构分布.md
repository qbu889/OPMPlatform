# OPMPlatform 项目文件结构分布
```
OPMPlatform/
├── templates/                  # 前端页面模板目录
│   ├── base.html               # 基础页面模板（父模板，未展示完整内容）
│   ├── document_formatter.html # 文档格式化操作页面
│   └── format_check.html       # 文档格式检查结果展示页面
├── utils/                      # 核心工具函数目录（业务逻辑层）
│   ├── demo_generator.py       # 按FPA规范生成功能需求Word文档
│   ├── document_formatter.py   # 模板格式分析、文档格式化（基础/增强版）
│   ├── format_matcher.py       # 文档格式匹配、结构对比、匹配度计算
│   └── parse_source_doc.py     # 文档结构验证接口（待完善实现）
├── routes/                     # Web路由处理目录（请求入口层）
│   ├── document_routes.py      # 文档格式化请求处理（Flask蓝图）
│   ├── mdtoword.py             # Markdown转Word（结构化填充模板）
│   ├── word_to_md_routes.py    # Word转Markdown（标题/段落格式转换）
│   └── __init__.py             # 路由蓝图初始化（默认隐含）
├── uploads/                    # 文件上传/输出目录（业务数据/生成文件）
│   ├── fpa_output_filled.md    # FPA解析后的业务数据（故障管理场景）
│   ├── 监控综合应用事件全景优化_功能需求.md # 功能需求原始MD文档
│   └── [临时文件]              # 上传源文档、格式化后输出文档（动态生成）
├── config.py                   # 项目配置文件（隐含，如上传目录、日志配置）
└── __init__.py                 # 项目初始化（隐含，Flask应用/蓝图注册）
```

## 各目录/文件核心功能说明
### 1. templates/（前端页面层）
| 文件名称                | 核心功能                                                                 |
|-------------------------|--------------------------------------------------------------------------|
| document_formatter.html | 文档格式化操作页：模板状态检测、待格式化文档上传、格式化结果展示/下载     |
| format_check.html       | 格式检查页：待检查文档上传、匹配度计算结果展示、格式要求说明、错误提示   |

### 2. utils/（核心业务逻辑层）
| 文件名称                | 核心功能                                                                 |
|-------------------------|--------------------------------------------------------------------------|
| demo_generator.py       | 基于FPA数据生成合规Word文档：剔除ILF/EIF项、多级标题样式控制、业务字段填充 |
| document_formatter.py   | 解析模板格式规则，提供基础/增强版文档格式化能力（标题/编号/段落样式适配） |
| format_matcher.py       | 解析文档结构（标题/字段）、对比模板与源文档、计算匹配度（标题/字段维度） |
| parse_source_doc.py     | 预留文档结构验证接口，待实现源文档（标题/表格）合规性校验逻辑             |

### 3. routes/（Web请求处理层）
| 文件名称                | 核心功能                                                                 |
|-------------------------|--------------------------------------------------------------------------|
| document_routes.py      | 接收文档格式化POST请求，校验模板/上传文件，调用格式化工具并返回结果       |
| mdtoword.py             | 将Markdown解析为结构化块（标题/列表/表格），填充至Word模板指定位置       |
| word_to_md_routes.py    | 将Word文档的标题/段落转换为Markdown格式（按Heading层级生成MD语法）        |

### 4. uploads/（数据存储层）
- 存储业务相关的原始MD文档、FPA解析后的数据文件（集中故障管理系统场景）；
- 动态存储用户上传的待处理Word文档、格式化/生成后的输出文档（临时文件）；
- 示例文件为运维类业务的功能需求数据，是文档生成的核心业务数据源。
