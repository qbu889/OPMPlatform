# 排班系统设计文档

## 📋 系统概述

排班系统是一个基于规则的自动化排班解决方案，支持日常和节假日两种不同排班模式，具备请假管理、人员轮换、冲突检测等功能。

### 1. 日期类型判断

- 若表中无记录，则根据周末判断（周六、周日为节假日）

```python
    # 如果有记录：is_working_day = 1 为日常，0 为节假日
    # 如果无记录：周末为节假日，工作日为日常
```

### 2. 节假日和日常排班的规则
```python
1. 从配置表查询出当前的人员名单的排班顺序，日常排班有日常排班配置人员顺序，节假日有节假日排班有配置人员，互为独立。
2. 排班的时候获取到第一位人员，并且查询请假表，没有请假则进行排班，有请假则取后面的人员直至取到人员，如果都请假了，默认排核心人员，并且添加备注（数据库表roster需要增加备注字段，有备注的时候进行备注，比如本来排A因A请假调整调整排班对象，或者因当前无可排人员默认排核心人员）；
3. 假如节假日顺序：曾婷婷、陈伟强、郑晨昊、吴绍烨、林子旺，排班的时候：排了曾婷婷，那么可排人员就更新为：陈伟强、郑晨昊、吴绍烨、林子旺
4. 再排班的时候：依次去减少已排班的人员，直至无人可排；
5. 无人可排的时候默认排核心人员；
6. 排班的时候无论是节假日还是日常排班，都需要考虑前一天8:0012:00或8:009:00时段有排过的人员，如果今天排到了在当前可排人员循环中延后排班。
7. 考虑今天是节假日还是日常，如果日常要查看日常的可排人员有谁，循环匹配这个人员是否请假、是否前天已排班；如果是节假日要查看日常节假日的可排人员有谁，循环匹配这个人员是否请假、是否前天已排班；
```
#### 3. 排班特殊考虑
- **请假过滤**: 排班当天的人员请假，则在当前循环中延后排班
- **默认兜底**: 无可用人员时默认安排核心人员

## 🎯 核心算法流程

### 排班生成主流程
```python
def generate_roster(self, start_date: date, end_date: date):
    1. 初始化轮换配置
    2. 遍历每一天：
       a. 判断日期类型（日常/节假日）
       b. 生成对应排班数据
       c. 写入数据库
       d. 更新轮换索引
```

### 日常排班生成
```python
def _get_daily_roster(self, target_date: date):
    1. 获取当日请假人员
    2. 生成8:00～9:00和18:00～21:00排班（同一人）
    3. 生成9:00～12:00排班（主辅班制）
    4. 生成13:30～18:00排班（全员参与）
```

### 节假日排班生成
```python
def _get_holiday_roster(self, target_date: date):
    1. 获取当日请假人员
    2. 从轮换队列中选择可用人员
    3. 为所有节假日时段安排同一人
    4. 更新轮换索引
```

## 🗄️ 数据库设计

### 主要数据表

#### 1. 排班表 (roster)
```sql
CREATE TABLE roster (
    id INT AUTO_INCREMENT PRIMARY KEY,
    date DATE NOT NULL,             # 排班日期
    time_slot VARCHAR(20) NOT NULL, # 时间段
    staff_name VARCHAR(50) NOT NULL,# 员工姓名
    is_main BOOLEAN,                # 是否为主班
    rotation_index INT              # 轮换索引
);
```

#### 2. 节假日配置表 (holiday_config)
```sql
CREATE TABLE holiday_config (
    id INT AUTO_INCREMENT PRIMARY KEY,
    holiday_date DATE NOT NULL,     # 节假日日期
    is_working_day BOOLEAN,         # 是否为工作日
    description VARCHAR(100)        # 描述
);
```

#### 3. 轮换配置表 (rotation_config)
```sql
CREATE TABLE rotation_config (
    time_slot_type VARCHAR(20),     # 时段类型
    rotation_order TEXT,            # 轮换顺序
    current_index INT               # 当前索引
);
```
