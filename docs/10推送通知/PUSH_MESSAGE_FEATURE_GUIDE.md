# 推送消息生成功能说明

## 功能概述

在 Kafka 消息生成器页面中，新增了【生成推送消息】按钮，可以根据生成的 FP 值快速创建推送消息。

## 使用流程

1. **访问 Kafka 消息生成器页面**
   - 打开浏览器访问：`http://127.0.0.1:5001/kafka-generator/`

2. **生成 Kafka 消息**
   - 粘贴 ES 数据中的 `_source` JSON 内容
   - 点击【生成 Kafka 消息】按钮
   - 等待系统处理并显示生成结果

3. **生成推送消息**
   - 在生成结果页面，点击【生成推送消息】按钮（蓝色按钮，带纸飞机图标）
   - 系统会自动弹出推送消息窗口

4. **查看和复制推送消息**
   - 弹窗中会显示：
     - FP 值（自动填充）
     - 事件时间（自动填充）
     - 激活状态（固定为 3）
     - 生成的推送消息 JSON
   - 点击【复制】按钮可以一键复制推送消息

## 推送消息格式

生成的推送消息格式如下：

```json
{
  "ACTIVE_STATUS": "3",
  "CFP0_CFP1_CFP2_CFP3": "1775094102_7429343158_6711009994_1989452915_54905",
  "EVENT_TIME": "2026-04-02 11:08:34",
  "FP0_FP1_FP2_FP3": "1775094102_7429343158_6711009994_1989452915_54905"
}
```

### 字段说明

- **ACTIVE_STATUS**: 激活状态，固定值为 "3"
- **CFP0_CFP1_CFP2_CFP3**: 清除告警指纹，与 FP 值保持一致
- **EVENT_TIME**: 事件时间，使用当前系统时间或从 Kafka 消息中提取的时间
- **FP0_FP1_FP2_FP3**: 告警指纹，从生成的 Kafka 消息中提取

## 技术实现

### 前端实现

- 新增【生成推送消息】按钮
- 新增推送消息弹窗（Modal）
- JavaScript 函数 `generatePushMessage()` 用于生成推送消息
- JavaScript 函数 `copyPushMessage()` 用于复制推送消息

### 后端实现

- 新增 API 路由：`POST /kafka-generator/generate-push-message`
- 接收参数：
  - `fp_value`: FP 值（必填）
  - `event_time`: 事件时间（可选，默认使用当前时间）
  - `active_status`: 激活状态（可选，默认为 "3"）
- 返回格式化的推送消息 JSON

## 测试方法

```bash

```

## 注意事项

1. 必须先生成 Kafka 消息后才能使用【生成推送消息】功能
2. 如果 Kafka 消息中没有 FP 值，系统会提示错误
3. 推送消息可以直接用于 Kafka 推送或其他系统集成

## 常见问题

**Q: 为什么点击【生成推送消息】按钮没有反应？**
A: 请确保已经成功生成了 Kafka 消息，并且结果中包含 FP0_FP1_FP2_FP3 字段。

**Q: 推送消息的时间是如何确定的？**
A: 默认使用 Kafka 消息中的 EVENT_TIME 字段，如果没有则使用当前系统时间。

**Q: ACTIVE_STATUS 可以修改吗？**
A: 当前版本固定为 "3"，如需修改可以联系开发人员调整。
