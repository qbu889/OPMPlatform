## 一、项目概述

### 1.1 背景
监控综合应用平台定期进行功能升级，每次升级需要生成标准化的软件装载记录表。当前升级申请以 Markdown/TXT 文本格式提交，人工整理为 Excel 表格效率低且易出错。本工具旨在自动化完成从文本内容到结构化 Excel 报表的转换。

### 1.2 目标
- **输入**: `.md` 或 `.txt` 格式的升级申请文档
- **输出**: 包含 4 个 Sheet 的标准化 Excel 文件（`.xlsx`）
- **核心能力**: 自动解析升级记录、计算版本号、生成 Excel 公式

### 1.3 适用范围
适用于所有遵循"监控综合应用平台 YYYY年M月D日功能升级申请"格式的升级申请文档。

## 二、输入规范

### 2.1 文件格式
支持以下格式的文件作为输入：
- `.md` (Markdown)
- `.txt` (纯文本)

### 2.2 内容结构要求
输入文档需包含若干条**升级申请记录**，每条记录的格式如下：

```
监控综合应用平台YYYY年M月D日功能升级申请（或服务迁移/安全加固）
[申请人列表]
Hi [称呼]

[升级说明正文]

监控综合应用值班 [电话号码]

[操作人员列表：姓名 电话]

一、升级功能如下：
[功能点列表，可选]
```

### 2.3 关键信息提取规则

| 字段 | 提取方式 | 示例 |
|------|----------|------|
| **升级日期** | 从标题"YYYY年M月D日"中提取 | `2026年3月11日` → `2026/3/11` |
| **升级类型** | 从标题中提取关键字 | `功能升级` / `服务迁移` / `安全加固` |
| **申请标题** | 拼接生成 | `监控综合应用平台2026年3月11日功能升级申请` |
| **装载日期** | 升级日期 + `22:00`（固定时间） | `2026/3/11 22:00` |
| **操作类型** | 根据升级类型映射 | `功能升级/服务迁移` → `更新`; `安全加固` → `安全加固` |

### 2.4 软件名称匹配规则
- 系统内置**软件名称关键字库**，用于将升级记录中的功能描述匹配到具体软件
- 关键字库支持**动态增删改查**（通过前端界面管理）
- 匹配优先级：精确匹配 > 模糊匹配 > 默认软件

### 2.5 示例输入（contentToChange.md）
```
监控综合应用平台2026年3月11日功能升级申请
何熙<13405905086@139.com>
叶芳彪<13799343810@139.com>
...

Hi 芳彪

监控综合应用平台申请3月11日晚进行补丁功能升级...

一、升级功能如下：
1、IVR督办增加保存实际拨打的状态
2、延期审批、挂起审批增加环节时限...
```

## 三、输出规范（Excel 结构）

### 3.1 Excel 文件整体结构
生成的 Excel 文件包含 **4 个 Sheet**：

| Sheet 名称 | 说明 | 行数（含表头） |
|-----------|------|---------------|
| `202603～5月` | 主表：装载记录汇总 | 动态（根据输入生成） |
| `版本号计算` | 版本计算公式表（含 Excel 公式） | 动态（历史+新增） |
| `初始版本号` | 软件基准版本参考表 | 固定（关键字库数量） |
| `软件名称和目标IP` | 软件-IP映射参考表 | 固定（关键字库数量） |

### 3.2 Sheet 1: `202603～5月`（主表）

**列定义（11 列）**：

| 列号 | 列名 | 数据来源 | 说明 |
|------|------|----------|------|
| A | 序号 | 自动生成 | 从 1 开始递增 |
| B | 软件名称 | 关键字匹配 | 从输入内容中提取并匹配 |
| C | 版本号 | 公式计算 | `=P列`（引用版本号计算Sheet） |
| D | 装载日期 | 日期提取 | `YYYY/M/D 22:00` 格式 |
| E | 操作类型 | 固定值 | `更新` / `安全加固` / `迁移` |
| F | 目标名称 | 空值预留 | 当前为空，预留扩展 |
| G | 目标IP | 软件-IP映射表 | `=VLOOKUP(B列, 软件名称和目标IP!$B:$F, 4, FALSE)` |
| H | 操作人员 | 软件-IP映射表 | `=VLOOKUP(B列, 软件名称和目标IP!$B:$F, 5, FALSE)` |
| I | 操作验证 | 固定值 | `正常` |
| J | 验证人员 | 软件-IP映射表 | `=VLOOKUP(B列, 软件名称和目标IP!$B:$F, 6, FALSE)` |
| K | 备注 | 申请标题拼接 | `="监控综合应用平台"&YEAR(D列)&"月"&DAY(D列)&"日功能升级申请"` |

**行生成规则**：
- 每条升级记录中，根据日期匹配对应的软件列表
- 3月仅包含 `智能调服务`（历史约定）
- 4月及之后包含 `省内工单服务` + `智能调服务`

### 3.3 Sheet 2: `版本号计算`（核心公式表）

**列定义（16 列）**：

| 列号 | 列名 | 公式/数据来源 | Excel 公式 |
|------|------|-------------|-----------|
| A | 序号 | 自动生成 | `=ROW()-1` |
| B | 软件名称 | 从输入提取 | 文本值 |
| C | 版本号 | 从输入提取 | 文本值（升级时声明的版本） |
| D | 装载日期 | 日期提取 | 日期值 |
| E | 操作类型 | 固定值 | `更新` / `安全加固` / `迁移` |
| F | 目标名称 | 空值预留 | 文本值 |
| G | 目标IP | 软件-IP映射表 | `=VLOOKUP(B列, 软件名称和目标IP!$B:$F, 4, FALSE)` |
| H | 操作人员 | 软件-IP映射表 | `=VLOOKUP(B列, 软件名称和目标IP!$B:$F, 5, FALSE)` |
| I | 操作验证 | 固定值 | `正常` |
| J | 验证人员 | 软件-IP映射表 | `=VLOOKUP(B列, 软件名称和目标IP!$B:$F, 6, FALSE)` |
| K | 备注 | 申请标题拼接 | `="监控综合应用平台"&YEAR(D列)&"月"&DAY(D列)&"日"&E列&"申请"` |
| L | 迭代次数 N | 公式计算 | `=COUNTIF($B$2:B列,B列)-1` |
| M | 首次初始版本 | VLOOKUP查询 | `=IFERROR(VLOOKUP(B列,初始版本号!$A:$B,2,0),"未匹配软件")` |
| N | 计算小数 | 公式计算 | `=IFERROR(LEFT(M列,FIND(".",M列)-1)+MID(M列,FIND(".",M列)+1,FIND(".",M列,FIND(".",M列)+1)-FIND(".",M列)-1)/10+RIGHT(M列,LEN(M列)-FIND(".",M列,FIND(".",M列)+1))/100,0)` |
| O | 版本计算 | 公式计算 | `=IFERROR(N列+L列*0.01,0)` |
| P | 标准版本号 | 公式计算 | `=IF(O列=0,"无版本",INT(O列)&"."&INT(MOD(O列,1)*10)&"."&MOD(INT(MOD(O列,1)*10),10))` |

**版本号计算逻辑详解**：
```
迭代次数 N = COUNTIF(该软件在此Sheet中已出现的次数) - 1

计算小数 = 主版本号 + (次版本号 / 10) + (修订号 / 100)
         = INT(初始版本.主.次.修) + (INT(初始版本.主.次) / 10) + (修订号 / 100)

版本计算 O = N + L * 0.01
           = 迭代次数 + 计算小数 * 0.01

标准版本号 P = IF(O=0, "无版本", INT(O).INT(MOD(O,1)*10).MOD(INT(MOD(O,1)*10),10))
```

**版本号计算示例**：
| 软件 | 初始版本 | 迭代次数 N | 计算小数 L | O = N+L*0.01 | 标准版本号 P |
|------|---------|-----------|-----------|-------------|-------------|
| 智能调服务 | 1.1.7 | 5 | 1.17 | 5.1700 | 1.2.2 |
| 省内工单服务 | 1.4.6 | 8 | 1.46 | 8.1460 | 1.5.4 |

### 3.4 Sheet 3: `初始版本号`（基准参考表）

**列定义（2 列）**：

| 列名 | 说明 |
|------|------|
| 软件名称 | 软件关键字（与输入内容匹配的关键字） |
| 初始化基准版本 | 该软件的初始版本号（如 `1.4.6`） |

**要求**:
- 支持通过前端界面进行增、删、改、查操作
- 新增的软件名称自动同步到 `软件名称和目标IP` Sheet

### 3.5 Sheet 4: `软件名称和目标IP`（映射参考表）

**列定义（5 列）**：

| 列名 | 说明 |
|------|------|
| 序号 | `=ROW()-1`（自动生成） |
| 软件名称 | 软件关键字 |
| 目标IP | 该软件部署的目标 IP 列表（逗号分隔） |
| 操作人员 | 默认操作人员 |
| 版本号 | 当前最新版本号（参考值） |

**要求**:
- 支持通过前端界面进行增、删、改、查操作
- 与 `初始版本号` Sheet 共享软件名称关键字库

## 四、功能需求

### 4.1 核心功能

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F1 | 文件上传 | 支持上传 `.md` / `.txt` 格式文件 | P0 |
| F2 | 内容解析 | 从文件中提取升级申请记录（日期、类型、功能点） | P0 |
| F3 | 软件匹配 | 将升级记录中的功能描述与关键字库进行匹配，确定涉及的软件 | P0 |
| F4 | 版本号计算 | 按照公式链（N→L→O→P）自动计算标准版本号 | P0 |
| F5 | Excel 生成 | 生成包含 4 个 Sheet 的 `.xlsx` 文件，含 Excel 公式 | P0 |
| F6 | 文件下载 | 生成的 Excel 文件供用户下载 | P0 |
| F7 | 关键字库管理 | 前端界面管理"软件名称和目标IP"及"初始版本号"（增删改查） | P1 |
| F8 | 历史数据保留 | `版本号计算` Sheet 保留历史记录，新增记录追加到末尾 | P1 |

### 4.2 关键字库管理（F7）

**软件名称和目标IP管理**:
- 列表展示所有已配置的软件及其目标 IP、操作人员、版本号
- 支持新增：输入软件名称、目标IP、操作人员、版本号
- 支持编辑：修改已有软件的任意字段
- 支持删除：移除不需要的软件配置

**初始版本号管理**:
- 列表展示所有软件的基准版本
- 支持新增、编辑、删除

### 4.3 解析规则细化（F2）

**升级申请标题正则**:
```regex
监控综合应用平台(\d{4})年(\d+)月(\d+)日(功能升级|服务迁移|安全加固)申请
```

**日期修正规则**:
- 原始文档中"3月2日"实际对应"3月11日"（历史修正）
- 日期格式统一转换为 `YYYY/M/D`

**软件匹配策略（F3）**:
1. **精确匹配**: 升级功能描述中直接出现软件名称 → 命中
2. **关键字匹配**: 根据预设的功能-软件映射关系推断涉及的软件
3. **日期规则**: 
   - 3月 → 仅 `智能调服务`
   - 4月及之后 → `省内工单服务` + `智能调服务`
4. **可扩展**: 支持通过前端配置更多匹配规则

## 五、非功能需求

### 5.1 性能要求
- 文件上传解析时间 ≤ 3 秒（常规文档，< 500行）
- Excel 生成时间 ≤ 5 秒

### 5.2 兼容性要求
- 支持 Python 3.8+
- 前端兼容 Chrome / Edge / Firefox 最新两个版本

### 5.3 安全要求
- 上传文件类型限制为 `.md` / `.txt`
- 文件大小限制 ≤ 5MB
- 文件名使用 `secure_filename` 处理

### 5.4 可维护性
- 软件匹配关键字库与解析逻辑解耦，便于扩展新软件
- Excel 公式生成逻辑独立模块，便于调整公式

## 七、数据流图

```
用户上传 .md/.txt 文件
        │
        ▼
   后端接收文件
        │
        ▼
   正则解析升级申请记录
   (提取日期、类型、标题)
        │
        ▼
   软件名称匹配（关键字库）
   (精确匹配 + 日期规则)
        │
        ▼
   版本号计算（公式链）
   (N → L → O → P)
        │
        ▼
   生成 Excel（4个Sheet）
   (含公式引用)
        │
        ▼
   返回文件供用户下载
```

## 九、附录

### 9.1 版本号计算公式速查

| 列 | 公式名称 | 核心逻辑 |
|----|---------|---------|
| L (迭代次数) | `COUNTIF` | 统计该软件在表中出现次数 -1 |
| M (初始版本) | `VLOOKUP` | 从"初始版本号"Sheet查询 |
| N (计算小数) | `LEFT/MID/RIGHT` | 解析"主.次.修"为小数 |
| O (版本计算) | `N + L*0.01` | 迭代次数 + 小数×0.01 |
| P (标准版本) | `INT/MOD` | 小数还原为"主.次.修"格式 |
