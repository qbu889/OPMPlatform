# 监控综合应用平台功能升级申请文档解析工具需求文档

## 一、项目概述

本工具用于将非结构化的监控综合应用平台功能升级申请文档（Markdown 格式）解析并转换为结构化的 Excel 表格，便于统计、查询和分析。

## 二、输入格式

### 2.1 文档结构
- 文件类型：Markdown (.md)
- 内容包含多个版本的升级申请记录

### 2.2 每份升级申请的组成部分
1. **标题**：监控综合应用平台 XXXX 年 XX 月 XX 日功能升级申请
2. **相关人员**：姓名 + 邮箱格式的联系信息
3. **升级说明**：具体的升级时间、影响范围描述
4. **值班信息**：监控综合应用值班电话
5. **联系人列表**：姓名 + 电话号码
6. **升级功能列表**：编号的功能点描述

## 三、输出格式（Excel 表格）

### 3.1 表格结构
| 序号 | 升级日期 | 申请类型 | 升级时间窗口 | 影响范围 | 值班电话 | 联系人姓名 | 联系电话 | 功能分类 | 具体功能描述 |
|------|----------|----------|--------------|----------|----------|------------|----------|----------|--------------|

### 3.2 字段说明
1. **序号**：自动生成的序号（从 1 开始）
2. **升级日期**：从标题中提取的日期，格式 YYYY-MM-DD
3. **申请类型**：如"功能升级"、"漏洞修复"、"服务迁移"、"安全加固"等
4. **升级时间窗口**：如"22 点至次日 5 点"，若无明确时间则填"无"
5. **影响范围**：升级期间受影响的功能或系统
6. **值班电话**：监控综合应用值班电话
7. **联系人姓名**：从文档中提取的所有联系人姓名（用分号分隔）
8. **联系电话**：对应的电话号码（用分号分隔）
9. **功能分类**：如"IVR 督办"、"审批流程"、"界面优化"、"接口优化"、"安全加固"等
10. **具体功能描述**：详细的功能点描述

## 四、解析规则

### 4.1 日期提取
- 从标题中提取"XXXX年 XX月 XX日"格式，转换为 YYYY-MM-DD

### 4.2 联系人提取
- 识别姓名 + 邮箱格式：`姓名<邮箱>`
- 识别姓名 + 电话格式：`姓名   电话`

### 4.3 功能分类
- 根据关键词自动识别功能类型：
  - "IVR" → IVR 督办
  - "审批" → 审批流程优化
  - "PC 端"/"APP 端" → 界面优化
  - "接口" → 接口优化
  - "漏洞"/"安全" → 安全加固
  - "迁移" → 服务迁移

### 4.4 时间窗口提取
- 识别"XX 点至 XX 点"格式的时间段

## 五、技术实现要求

### 5.1 开发语言
- Python 3.x

### 5.2 依赖库
- openpyxl / xlsxwriter (Excel 生成)
- re (正则表达式解析)

### 5.3 文件路径
- 输入：`docs/tools/contentToExcel/contentToChange.md`
- 输出：`docs/tools/contentToExcel/shengchengbiaoge.xlsx`

## 六、处理流程

1. 读取 Markdown 源文件
2. 使用正则表达式分割多个升级申请记录
3. 对每条记录解析：
   - 提取标题和日期
   - 提取联系人信息
   - 提取升级说明和时间窗口
   - 解析功能列表
4. 生成 Excel 表格并保存
