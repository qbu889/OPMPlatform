# 调整因子计算器使用说明

## 功能概述

调整因子计算器是一个用于 FPA（功能点分析）的辅助工具，根据项目的具体条件动态计算调整因子。该工具将 Excel 表格中的 9 大类调整因子数据模块化展示，并提供实时计算功能。

## 访问路径

- **首页入口**：系统首页 → 工具菜单 → 调整因子计算器
- **直接访问**：`/adjustment-calc/page`
- **管理后台**：`/adjustment/page`（调整因子数据管理）

## 功能模块

### 第一部分：项目基本信息（动态计算）

这部分内容根据用户选择自动计算调整因子值，包含以下 7 个维度：

#### 1. 规模计数时机
- **估算早期**（概算、预算阶段）：1.39
- **估算中期**（投标、项目计划阶段）：1.21
- **估算晚期**（需求分析阶段）：1.10
- **项目完成**（项目交付后及运维阶段）：1.00

#### 2. 应用类型
- 业务处理：1.0
- 应用集成：1.2
- 科技：1.2
- 多媒体：1.5
- 智能信息：1.7
- 基础软件/支撑软件：1.7
- 通信控制：1.9
- 流程控制：2.0

#### 3. 质量特性（4 个子项）

**分布式处理**
- 没有明示对分布式处理的需求事项：-1
- 通过网络进行客户端/服务器及网络基础应用分布处理和传输：0
- 在多个服务器及处理器上同时相互执行计算机系统中的处理功能：1

**性能**
- 没有明示对性能的特别需求事项或活动，因此提供基本性能：-1
- 应答时间或处理率对高峰时间或所有业务时间来说都很重要：0
- 为满足性能需求事项，要求设计阶段开始进行性能分析：1

**可靠性**
- 没有明示对可靠性的特别需求事项或活动，因此提供基本的可靠性：-1
- 发生故障时可轻易修复，带来一定不便或经济损失：0
- 发生故障时很难复，发生重大经济损失或有生命危害：1

**多重站点**
- 在相同用途的硬件或软件环境下运行：-1
- 在用途类似的硬件或软件环境下运行：0
- 在不同用途的硬件或软件环境下运行：1

#### 4. 开发语言
- C 及其他同级别语言/平台：1.2
- JAVA、C++、C#及其他同级别语言/平台：1.0
- PowerBuilder、ASP 及其他同级别语言/平台：0.8

#### 5. 开发团队背景
- 为本行业（政府）开发过类似的软件：0.8
- 为其他行业开发过类似的软件，或为本行业（政府）开发过不同但相关的软件：1.0
- 没有同类软件及本行业（政府）相关软件开发背景：1.2

#### 6. 重用程度
- 高（代码复用程度高）：0.33
- 中（代码复用程度中等）：0.67
- 低（代码复用程度低）：1.00

#### 7. 修改类型
- 新增（本期项目新增的功能模块）：1.00
- 修改（修改往期项目的功能模块）：0.80
- 删除（删除往期项目的功能模块）：0.20

### 第二部分到第九部分：参考数据展示

这些部分展示固定的参考数据，帮助用户了解各项调整因子的标准值。

- **第二部分**：应用类型参考表
- **第三部分**：质量特性判断标准
- **第四部分**：规模变更调整因子 (CF)
- **第五部分**：开发语言参考表
- **第六部分**：开发团队背景参考表
- **第七部分**：重用程度调整系数
- **第八部分**：修改类型调整系数
- **第九部分**：软件开发计价模型

## 使用方法

### 步骤 1：填写项目信息
在第一部分中选择项目的各项参数：
1. 选择规模计数时机
2. 选择应用类型
3. 选择质量特性的 4 个子项（分布式处理、性能、可靠性、多重站点）
4. 选择开发语言
5. 选择开发团队背景
6. 选择重用程度
7. 选择修改类型

### 步骤 2：点击计算
填写完所有必填项后，点击"计算总调整因子"按钮。

### 步骤 3：查看结果
系统会显示：
- 各项调整因子的具体值
- 详细计算过程
- 总调整因子（所有因子的乘积）

### 计算公式

**总调整因子 = 规模变更 × 应用类型 × 质量特性 × 开发语言 × 团队背景 × 重用程度 × 修改类型**

示例计算：
```
规模变更（估算中期）: 1.21
应用类型（业务处理）: 1.0
质量特性（-1 + -1 + -1 + -1）: -4
开发语言（JAVA）: 1.0
团队背景（本行业类似）: 0.8
重用程度（低）: 1.0
修改类型（新增）: 1.0

总调整因子 = 1.21 × 1.0 × (-4) × 1.0 × 0.8 × 1.0 × 1.0 = -3.872
```

## 技术实现

### 后端接口

#### 1. 获取计算器数据
```
GET /adjustment-calc/api/calculator-data
```
返回所有调整因子数据，按分类组织。

#### 2. 计算调整因子
```
POST /adjustment-calc/api/calculate
Content-Type: application/json

{
    "scale_timing": "估算中期",
    "application_type": "业务处理",
    "distributed_processing": "...",
    "performance": "...",
    "reliability": "...",
    "multi_site": "...",
    "language": "JAVA、C++、C#及其他同级别语言/平台",
    "team_background": "...",
    "reuse_level": "低",
    "change_type": "新增"
}
```

### 数据库表结构

数据存储在 `fpa_adjustment_factor` 表中：

```sql
CREATE TABLE fpa_adjustment_factor (
    id INT PRIMARY KEY AUTO_INCREMENT,
    factor_name VARCHAR(100),      -- 因子名称
    factor_category VARCHAR(50),   -- 因子分类
    option_name TEXT,              -- 选项名称
    score_value DECIMAL(10,4),     -- 分值
    formula TEXT,                  -- 公式
    display_order INT,             -- 显示顺序
    parent_id INT,                 -- 父级 ID
    created_at TIMESTAMP,          -- 创建时间
    updated_at TIMESTAMP           -- 更新时间
);
```

## 注意事项

1. **所有字段必填**：计算前必须填写所有下拉选择框
2. **质量特性计算**：4 个子项（分布式处理、性能、可靠性、多重站点）的分值相加
3. **数据准确性**：所有调整因子值均来自 Excel 原始表格
4. **公式保留**：第一部分的选择项保留了原始 Excel 公式作为参考
5. **数据管理**：可通过 `/adjustment/page` 管理后台修改调整因子数据

## 常见问题

### Q1: 为什么质量特性调整因子是负数？
A: 当选择"没有明示对...的特别需求事项"时，根据 Excel 公式计算结果为 -1，4 项相加可能得到负数。这是正常的评估结果。

### Q2: 如何修改调整因子的值？
A: 访问调整因子管理页面（`/adjustment/page`），可以进行增删改查操作。

### Q3: 计算结果不准确怎么办？
A: 检查是否所有字段都已正确选择，特别是质量特性的 4 个子项都需要选择。

### Q4: 数据从哪里来？
A: 数据来自 Excel 文件 `/test/fpa/调整因子.xlsx`，通过导入脚本写入数据库。
