# EsToExcel 使用指南

## 功能概述

EsToExcel 工具支持将 ES（Elasticsearch）查询结果转换为 Excel 文件，方便后续数据分析。

## 支持的输入格式

### 1. 文件上传模式
- **竖线分隔格式**：传统的 `|` 分隔文本
- **JSON 格式**：ES SQL 查询返回的 JSON 数据
- **ES SQL 表格格式**：`POST /_sql?format=txt` 返回的表格样式

### 2. 文本粘贴模式
直接粘贴 ES 查询结果到文本框，无需保存为文件。

## 核心功能

### ✅ 自动格式检测
系统会自动识别输入文件的格式并选择合适的解析方式。

### ✅ 时间格式转换
自动将 ISO 8601 格式转换为标准格式：
```
2026-04-16T10:04:21.000Z → 2026-04-16 10:04:21
```

### ✅ 字段中英文映射
勾选「映射中文字段名」后，Excel 表头将显示中文名称，基于数据库配置自动映射。

**示例：**
- `CREATION_EVENT_TIME` → `事件发现时间`
- `ORDER_ID` → `工单号`
- `EVENT_FP` → `事件流水号`

### ✅ 多文件格式支持
- **XLSX**（推荐）：现代 Excel 格式，支持更多数据和功能
- **XLS**（兼容旧版）：兼容达梦数据库迁移工具

## 使用步骤

### 方式一：文件上传

1. 点击「文件上传」Tab
2. 拖拽或选择 `.txt` 文件（支持多个文件）
3. 点击「上传全部文件」
4. 选择是否勾选「映射中文字段名」
5. 选择 Excel 格式（XLSX 或 XLS）
6. 点击「预览数据」查看前 10 条记录
7. 点击「转换为 Excel」生成文件
8. 点击「下载 Excel 文件」保存到本地

### 方式二：文本粘贴

1. 点击「文本粘贴」Tab
2. 粘贴 ES 查询结果（支持三种格式）
3. 选择是否勾选「映射中文字段名」
4. 选择 Excel 格式
5. 点击「处理并生成 Excel」
6. 点击「下载 Excel 文件」保存到本地

## 典型使用场景

### 场景 1：ES SQL 查询结果导出

```sql
POST /_sql?format=txt
{
  "query": """
    SELECT 
      "CREATION_EVENT_TIME",
      "EVENT_FP",
      "ORDER_ID",
      "DISPATCH_INFO.DISPATCH_REASON"
    FROM "mw_em_master"
    WHERE 
      "CREATION_EVENT_TIME" >= '2026-04-16 00:00:00'
      AND "ORDER_ID" IS NOT NULL
    ORDER BY "CREATION_EVENT_TIME" DESC
  """
}
```

**操作步骤：**
1. 复制查询结果（包含表头和分隔线）
2. 在「文本粘贴」Tab 中粘贴
3. 勾选「映射中文字段名」
4. 点击「处理并生成 Excel」
5. 下载生成的 Excel 文件

### 场景 2：批量文件合并

如果需要将多个 ES 查询结果合并到一个 Excel 文件中：
1. 在「文件上传」Tab 中上传多个 `.txt` 文件
2. 确保所有文件的字段结构一致
3. 点击「转换为 Excel」
4. 系统会自动合并所有文件并生成一个 Excel

## 字段映射配置

字段映射关系维护在 CSV 文件中：
```
docs/EsToExcel/dm_import_2604101438_converted.csv
```

**CSV 格式：**
- 第一行：英文字段名
- 第二行：中文字段名

如需添加新的字段映射，编辑此文件即可。

## 注意事项

1. **文件格式**：仅支持 `.txt` 格式的文件上传
2. **编码支持**：自动检测 UTF-8、GBK、GB2312 等编码
3. **字段一致性**：批量上传时，所有文件的字段必须一致
4. **时间格式**：只有包含 `T` 和 `Z` 的 ISO 8601 格式会被自动转换
5. **空值处理**：`null` 值会自动转换为空字符串

## 技术架构

- **前端**：Vue 3 + Element Plus
- **后端**：Flask Blueprint
- **数据处理**：Pandas DataFrame
- **Excel 生成**：openpyxl (XLSX) / xlwt (XLS)

## 常见问题

### Q1: 为什么有些字段没有映射成中文？
A: 只有在 CSV 配置文件中定义的字段才会被映射。未配置的字段保持原样。

### Q2: 粘贴的文本格式不正确怎么办？
A: 确保粘贴的内容包含完整的表头和分隔线（对于 ES SQL 表格格式）。

### Q3: 如何查看完整的字段列表？
A: 点击「预览数据」按钮，可以查看前 10 条记录和所有字段名。

### Q4: XLS 和 XLSX 有什么区别？
A: 
- XLSX：现代格式，支持更多数据，推荐使用
- XLS：旧版格式，兼容达梦数据库迁移工具
