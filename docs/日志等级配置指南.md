# 日志等级配置指南

## 功能概述

本项目支持通过环境变量灵活控制日志输出级别，可以根据需要选择只输出错误日志、警告日志或所有调试信息。

## 配置方式

### 方式一：修改 .env 文件（推荐）

在项目根目录的 `.env` 文件中添加以下配置：

```bash
# 日志配置
# 日志级别：ERROR / WARNING / INFO / DEBUG
LOG_LEVEL=INFO

# 是否输出到控制台（默认：True）
LOG_TO_CONSOLE=True

# 是否输出到文件（默认：True）
LOG_TO_FILE=True
```

### 方式二：设置系统环境变量

在启动应用前设置环境变量：

**Linux/Mac:**
```bash
export LOG_LEVEL=ERROR
export LOG_TO_CONSOLE=True
export LOG_TO_FILE=True
python app.py
```

**Windows PowerShell:**
```powershell
$env:LOG_LEVEL="ERROR"
$env:LOG_TO_CONSOLE="True"
$env:LOG_TO_FILE="True"
python app.py
```

## 日志级别说明

### 1. ERROR（错误日志）
- **输出内容**：只输出错误信息
- **适用场景**：生产环境，只关注系统错误
- **示例配置**：
  ```bash
  LOG_LEVEL=ERROR
  ```

### 2. WARNING（警告日志）
- **输出内容**：错误信息 + 警告信息
- **适用场景**：生产环境，需要关注潜在问题
- **示例配置**：
  ```bash
  LOG_LEVEL=WARNING
  ```

### 3. INFO（信息日志）⭐ 默认
- **输出内容**：错误 + 警告 + 普通信息
- **适用场景**：开发环境、日常运维
- **示例配置**：
  ```bash
  LOG_LEVEL=INFO
  ```

### 4. DEBUG（调试日志）
- **输出内容**：所有日志（包括详细调试信息）
- **适用场景**：开发调试、问题排查
- **示例配置**：
  ```bash
  LOG_LEVEL=DEBUG
  ```

## 其他配置选项

### 控制台输出开关
```bash
# 不在控制台输出日志
LOG_TO_CONSOLE=False

# 在控制台输出日志（默认）
LOG_TO_CONSOLE=True
```

### 文件输出开关
```bash
# 不写入日志文件
LOG_TO_FILE=False

# 写入日志文件（默认）
LOG_TO_FILE=True
```

### 自定义日志格式
```bash
# 使用简化格式
LOG_FORMAT=%(asctime)s - %(name)s - %(levelname)s - %(message)s

# 使用详细格式（默认）
LOG_FORMAT=%(asctime)s | %(levelname)-8s | %(name)-40s | %(message)s
```

### 自定义日期格式
```bash
# 使用斜杠分隔
LOG_DATE_FORMAT=%Y/%m/%d %H:%M:%S

# 使用默认格式
LOG_DATE_FORMAT=%Y-%m-%d %H:%M:%S
```

## 使用示例

### 示例 1：生产环境（只输出错误）
```bash
LOG_LEVEL=ERROR
LOG_TO_CONSOLE=True
LOG_TO_FILE=True
```

### 示例 2：开发环境（输出所有信息）
```bash
LOG_LEVEL=DEBUG
LOG_TO_CONSOLE=True
LOG_TO_FILE=True
```

### 示例 3：仅文件日志（不输出到控制台）
```bash
LOG_LEVEL=INFO
LOG_TO_CONSOLE=False
LOG_TO_FILE=True
```

### 示例 4：临时调试（命令行设置）
```bash
# Linux/Mac
LOG_LEVEL=DEBUG python app.py

# Windows
set LOG_LEVEL=DEBUG && python app.py
```

## 日志文件位置

日志文件默认保存在项目根目录的 `logs/` 文件夹中：

```
wordToWord/
├── logs/
│   ├── app_20260320.log    # 当天的日志
│   ├── app_20260319.log    # 历史日志
│   └── ...
```

日志文件按日期命名，每天一个文件。

## 验证配置

可以通过以下方式验证日志配置是否生效：

1. **查看启动日志**：应用启动时会输出当前配置的日志级别
2. **运行测试脚本**：
   ```bash
   python test_log_levels.py
   ```
   该脚本会依次测试 ERROR、WARNING、INFO、DEBUG 四个级别的输出效果

## 常见问题

### Q: 修改配置后不生效？
A: 请确保重启了应用程序，环境变量在应用启动时读取。

### Q: 为什么有些模块的日志还是很多？
A: 某些第三方库（如 urllib3）有自己的日志配置，需要单独设置它们的级别。

### Q: 如何在代码中临时调整日志级别？
A: 可以在运行时动态设置：
```python
import logging
logging.getLogger('某个模块').setLevel(logging.ERROR)
```
