# **跨专业派单时限就短优化**

## **原始需求**

针对子事件是集客和家宽的跨专业关联事件派单场景，主事件任务处理时限按主子事件（子事件需符合派单规则及时延）时限最短的进行派

## **需求描述**

针对专业=“集客”或“家宽”的事件，若有触发“关联追单”，且追单成功的，触发工单/任务处理时限更新判断

主单的专业与当前子单专业不一致

判断主单处理时限分钟数（故障处理时限） < 当前关联追子单的处理时限分钟数（故障处理时限）

按照当前子单的处理时限分钟数 + 主单派发时间重新计算处理时限，更新工单及任务处理时限，包含故障受理/处理时限、T1受理/处理时限、T2受理/处理时限；

若处理时限有变更，且 当前任务处于“延期待审批”或“挂起待审批”状态，则自动驳回延期/挂起申请，驳回原因为“影响【当前子事件的专业】业务，需按业务要求时限处理。”

若处理时限有变更，且 当前任务处于“挂起”状态，则取消“挂起”状态、任务当前处理人变更为原“挂起申请”的人员/维护组（原先为个人分支的回退到人，原先为同组处理分支的回退到组）；同时调度作战室留痕，留痕标题=“调度指令”，留痕了内容=“影响【当前子事件的专业】业务，需按业务要求时限处理，撤销任务挂起，任务指派给【人员姓名（电话号码）】/【XXX维护组】”

任务处理时限更新后，更新当前任务的任务描述（针对每个子任务的最新一条任务描述）；同时任务调度作战室留痕，留痕标题=“调度指令”，留痕了内容=“影响【当前子事件的专业】业务，任务处理时限由【原任务处理时限】变更为【变更后任务处理时限】”

任务处理时限有变更时，需IVR通知任务当前处理人/组

IVR内容模板：【任务号】【任务名称】影响【当前子事件的专业】业务，变更处理时限为【任务当前处理时限】，请及时处理。

IVR通知对象：任务当前处理人 或 当前处理组 ；若有进行延期/挂起驳回、撤销挂起，则通知的是变更后的任务处理人/组。（若通知对象是组，则需要根据对应组的IVR排班表进行通知）
