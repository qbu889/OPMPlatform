# 停电引起油机状态告警派发任务流程优化

## 原始需求

       市电停电（固定油机）已恢复，但伴随油机状态告警未恢复，需基于油机状态告警派发动环停电任务

## 需求描述

1.  “事件标签规则”配置界面优化
    
    ![c573aa1d14928d0aa1f99f3054ba3252.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/NpQlK5jeAo0mZqDv/img/a39f1f51-00dc-40d1-8d2e-50b0fa0307d3.png)
    
    1.  设置事件标签值：增加“更新范围”字段，下拉框，可选项：主事件、子事件、主子事件；每个赋值字段单独配置
        
    2.  事件标签规则生效场景为“事件关联后”和“子事件派单后”的场景，该字段可选；若是其他场景，则默认选中“主事件”，并且不可编辑
        
2.  “事件标签”数据赋值优化
    
    1.  满足“事件标签规则”的数据赋值时，根据对应字段配置的“更新范围”确定更新子事件或主事件；兼容该字段为空的场景，对应更新主事件；
        
    2.  该字段针对事件标签规则生效场景为“事件关联后”和“子事件派单后”的场景，其他场景默认为更新主事件；针对子事件更新的，只更新当前子事件，不再对当前主事件的其他子事件进行处理；
        
    3.  功能上线后，历史规则数据对应字段的值配置为“主事件”
        
    
3.  “事件标签”规则配置
    
    1.  配置生效条件为“事件关联后”，主事件为停电事件，子事件为油机状态事件
        
    2.  设置事件标签字段：是否匹配智能调规则=1，智能调派发场景=10000（停电流程）
