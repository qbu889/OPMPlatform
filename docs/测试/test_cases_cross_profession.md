## 测试用例详细设计

### 一、正向测试用例 (Positive Test Cases)

#### TC-POS-001: 集客 + 家宽跨专业关联追单成功，时限更新

| 字段 | 内容 |
|------|------|
| **优先级** | P0 - 高优先级 |
| **前置条件** | 1. 主事件专业=集客<br>2. 子事件专业=家宽<br>3. 触发关联追单成功 |
| **测试步骤** | 1. 创建主事件（集客专业，故障处理时限=60 分钟）<br>2. 创建子事件（家宽专业，故障处理时限=120 分钟）<br>3. 触发关联追单并成功 |
| **预期结果** | 1. 主单处理时限更新为：子单处理时限 (120 分钟) + 主单派发时间<br>2. 工单及任务处理时限更新（故障受理/处理时限、T1 受理/处理时限、T2 受理/处理时限）<br>3. 任务描述更新为最新一条任务描述<br>4. 调度作战室留痕：'影响【家宽】业务，任务处理时限由【原任务处理时限】变更为【变更后任务处理时限】'<br>5. IVR 通知当前处理人/组：'变更处理时限为【任务当前处理时限】，请及时处理。' |

#### TC-POS-002: 家宽 + 集客跨专业关联追单成功，时限更新

| 字段 | 内容 |
|------|------|
| **优先级** | P0 - 高优先级 |
| **前置条件** | 1. 主事件专业=家宽<br>2. 子事件专业=集客<br>3. 触发关联追单成功 |
| **测试步骤** | 1. 创建主事件（家宽专业，故障处理时限=90 分钟）<br>2. 创建子事件（集客专业，故障处理时限=60 分钟）<br>3. 触发关联追单并成功 |
| **预期结果** | 1. 主单处理时限更新为：子单处理时限 (60 分钟) + 主单派发时间<br>2. 工单及任务处理时限更新（故障受理/处理时限、T1 受理/处理时限、T2 受理/处理时限）<br>3. 任务描述更新为最新一条任务描述<br>4. 调度作战室留痕：'影响【集客】业务，任务处理时限由【原任务处理时限】变更为【变更后任务处理时限】'<br>5. IVR 通知当前处理人/组 |

#### TC-POS-003: 主单时限 < 子单时限，需要更新

| 字段 | 内容 |
|------|------|
| **优先级** | P1 - 中优先级 |
| **前置条件** | 主事件处理时限=30 分钟，子事件处理时限=60 分钟 |
| **测试步骤** | 1. 创建主事件（集客专业，30 分钟）<br>2. 创建子事件（家宽专业，60 分钟）<br>3. 触发关联追单成功 |
| **预期结果** | 1. 判断主单处理时限 (30 分钟) < 子单处理时限 (60 分钟)<br>2. 按子单处理时限 + 主单派发时间重新计算<br>3. 更新所有相关时限字段 |

#### TC-POS-004: 主单时限 >= 子单时限，无需更新

| 字段 | 内容 |
|------|------|
| **优先级** | P1 - 中优先级 |
| **前置条件** | 主事件处理时限=120 分钟，子事件处理时限=60 分钟 |
| **测试步骤** | 1. 创建主事件（集客专业，120 分钟）<br>2. 创建子事件（家宽专业，60 分钟）<br>3. 触发关联追单成功 |
| **预期结果** | 1. 判断主单处理时限 (120 分钟) >= 子单处理时限 (60 分钟)<br>2. 不触发时限更新<br>3. 不触发驳回/撤销挂起操作 |
