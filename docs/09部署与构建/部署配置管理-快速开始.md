# 部署配置管理 - 快速开始指南

## 📋 概述

本功能实现了**动态配置管理**，所有部署相关参数（服务器IP、端口、路径等）都可以通过页面修改，无需改代码、无需重启服务。

---

## 🚀 快速开始

### 1. 初始化数据库表

```bash
# 在项目根目录执行
python scripts/init_deploy_config.py
```

输出示例：
```

✅ 表创建成功
📝 正在插入默认配置...
✅ 成功插入 16 条默认配置

📊 当前配置总数: 16

📋 配置示例:
   - remote_host: 8.146.228.47 (server)
   - remote_user: root (server)
   - backend_port: 5004 (server)
   ...

✅ 部署配置初始化完成！
```

### 2. 启动应用

```bash
# 启动后端
python app.py

# 启动前端（开发环境）
cd frontend
npm run dev
```

### 3. 访问配置管理页面

打开浏览器访问：
```
http://localhost:5173/deploy-config
```

或生产环境：
```
http://8.146.228.47:5173/deploy-config
```

## 🎨 界面说明

### 配置分类标签页

1. **🖥️ 服务器配置**
   - 远程服务器IP地址
   - SSH用户名和端口
   - 项目路径和备份目录

2. **🚀 部署配置**
   - Git分支
   - 超时时间
   - 是否启用自动备份

3. **💾 备份配置**
   - 最大备份数量
   - 备份保留天数

4. **📊 监控配置**
   - 日志查看行数
   - 健康检查间隔

### 操作按钮

- **保存**：保存单个配置的修改
- **测试**：测试SSH连接是否正常
- **保存所有修改**：批量保存所有修改
- **刷新配置**：重新从数据库加载配置
- **恢复默认配置**：重置为初始值

## 💡 使用示例

### 场景1：更换服务器IP

1. 进入"服务器配置"标签页
2. 找到 `remote_host` 配置项
3. 修改值为新的IP地址（如 `192.168.1.100`）
4. 点击"保存"按钮
5. 点击"测试"按钮验证连接

✅ 立即生效，无需重启服务！

### 场景2：调整备份策略

1. 进入"备份配置"标签页
2. 修改 `max_backup_count` 为 `10`（保留10个备份）
3. 修改 `backup_retention_days` 为 `60`（保留60天）
4. 点击"保存所有修改"

✅ 下次部署时自动应用新策略！

### 场景3：切换Git分支

1. 进入"部署配置"标签页
2. 修改 `git_branch` 为 `main` 或其他分支
3. 点击"保存"

✅ 下次部署将使用新分支！

## 🔧 API 使用

### 获取所有配置

```bash
curl -X GET "http://localhost:5004/api/deploy/config/list" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

响应：
```json
{
  "success": true,
  "configs": [
    {
      "config_key": "remote_host",
      "config_value": "8.146.228.47",
      "config_type": "string",
      "description": "远程服务器IP地址",
      "category": "server"
    }
  ],
  "categories": ["server", "deployment", "backup", "monitor"]
}
```

### 更新单个配置

```bash
curl -X POST "http://localhost:5004/api/deploy/config/update" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "config_key": "remote_host",
    "config_value": "192.168.1.100"
  }'
```

### 批量更新配置

```bash
curl -X POST "http://localhost:5004/api/deploy/config/batch-update" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "configs": [
      {"config_key": "remote_host", "config_value": "192.168.1.100"},
      {"config_key": "backend_port", "config_value": "5005"}
    ]
  }'
```

### 测试SSH连接

```bash
curl -X POST "http://localhost:5004/api/deploy/config/test-connection" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "remote_host": "8.146.228.47",
    "remote_user": "root",
    "ssh_port": 22
  }'
```

## ⚠️ 注意事项

### 1. 权限控制

配置管理页面**仅管理员可访问**，需要登录后才能操作。

### 2. 敏感信息

- `deploy_password` 等敏感配置在列表中显示为 `***`
- 编辑时可输入新值
- 数据库中明文存储（建议后续加密）

### 3. 配置验证

修改关键配置（如服务器IP）后，务必点击"测试"按钮验证连通性。

### 4. 备份建议

修改配置前建议先导出当前配置：

```bash
# 导出所有配置到JSON
curl -X GET "http://localhost:5004/api/deploy/config/list" \
  -H "Authorization: Bearer YOUR_TOKEN" > config_backup.json
```

## 🐛 常见问题

### Q1: 修改配置后不生效？

**A**: 检查以下几点：
1. 确认已点击"保存"按钮
2. 查看浏览器控制台是否有错误
3. 检查后端日志确认更新成功
4. 刷新页面重新加载配置

### Q2: SSH连接测试失败？

**A**: 可能原因：
1. 服务器IP或端口错误
2. SSH密钥未配置
3. 防火墙阻止连接
4. 服务器未启动SSH服务

解决步骤：
```bash
# 手动测试SSH连接
ssh root@8.146.228.47

# 检查SSH服务状态
systemctl status sshd
```

### Q3: 如何恢复默认配置？

**A**: 
1. 点击"恢复默认配置"按钮
2. 或者重新运行初始化脚本：
   ```bash
   python scripts/init_deploy_config.py
   ```
