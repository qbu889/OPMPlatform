# 部署管理中心使用指南

## 📋 概述

部署管理中心是一个Web化的部署管理界面，提供可视化的部署、备份、恢复和日志监控功能。

**访问地址**: http://192.168.124.11:5200/deploy-config

---

## ✨ 核心功能

### 1. 快速部署
- **适用场景**: 日常小改动，仅修改Python文件
- **特点**: 
  - 跳过Git操作
  - 不重新构建前端
  - 直接上传变更的后端文件
  - 可选择是否重启服务
- **耗时**: 10-30秒

### 2. 完整部署
- **适用场景**: 重大更新，包含前端修改
- **特点**:
  - Git提交并推送
  - 自动检测变更文件
  - 前端重新构建
  - 后端打包上传
  - Nginx配置更新
  - 服务重启
- **耗时**: 60-120秒

### 3. 创建备份
- **功能**: 手动创建当前版本的备份
- **排除项**: node_modules、.venv、__pycache__、*.pyc、logs/*.log
- **存储位置**: /project/backups/
- **命名格式**: wordToWord_backup_YYYYMMDD_HHMMSS.tar.gz

### 4. 恢复备份
- **功能**: 从历史备份版本恢复
- **操作流程**:
  1. 点击"恢复备份"按钮
  2. 选择要恢复的备份版本
  3. 确认恢复操作
  4. 系统自动停止服务、解压备份、重启服务
- **警告**: 此操作不可逆！

### 5. 重启服务
- **功能**: 仅重启后端Python服务
- **不影响**: Nginx配置、前端静态文件
- **用途**: 应用配置变更、代码热更新失败时

### 6. 实时日志监控
- **部署日志**: 实时显示部署过程中的每一步操作
- **服务器日志**: 
  - 后端日志 (backend.log)
  - Nginx访问日志
  - Nginx错误日志
- **特性**:
  - SSE (Server-Sent Events) 实时推送
  - 自动滚动到底部
  - 日志级别着色（info/success/warning/error）

### 7. 备份版本管理
- **列表展示**: 显示最近50个备份版本
- **信息包括**: 备份名称、时间、大小
- **操作**: 一键恢复到指定版本

## 🎯 使用流程

### 日常开发部署

```
1. 在本地完成代码修改
2. 访问 http://192.168.124.11:5200/deploy-config
3. 点击"快速部署"按钮
4. 查看实时日志确认部署进度
5. 等待部署完成提示
6. 刷新浏览器验证效果
```

### 重大版本发布

```
1. 完成所有功能开发和测试
2. 访问部署管理中心
3. 点击"创建备份"（可选，建议先备份）
4. 点击"完整部署"按钮
5. 观察实时日志
6. 等待所有步骤完成
7. 全面测试系统功能
```

### 版本回滚

```
1. 发现新版本有问题
2. 访问部署管理中心
3. 点击"恢复备份"按钮
4. 选择合适的备份版本
5. 确认恢复操作
6. 等待系统自动恢复
7. 验证回滚成功
```

## 📊 界面说明

### 部署状态卡片
- **当前步骤**: 显示正在执行的操作
- **进度条**: 可视化部署进度 (0-100%)
- **上次部署**: 最后一次部署的时间
- **状态标签**: 成功/失败/未知

### 操作按钮区域
五个主要操作按钮，部署进行中时会自动禁用防止冲突。

### 实时部署日志
- 黑色背景，等宽字体
- 不同级别用不同颜色区分：
  - 🔵 蓝色: INFO
  - 🟢 绿色: SUCCESS
  - 🟡 黄色: WARNING
  - 🔴 红色: ERROR
- 自动滚动到最新日志

### 服务器日志查看器
- 可切换查看不同类型的日志
- 显示最后100行
- 支持手动刷新

### 备份版本列表
- 表格形式展示所有备份
- 按时间倒序排列（最新的在前）
- 每行都有"恢复"按钮

## 🔧 技术实现

### 后端架构
- **Flask Blueprint**: `/deploy-config` 前缀
- **多线程**: 部署任务在后台线程执行，不阻塞API响应
- **SSE**: Server-Sent Events 实现实时日志推送
- **SSH/SCP**: 远程命令执行和文件传输

### 前端技术
- **Vue 3 Composition API**: 响应式数据管理
- **Element Plus**: UI组件库
- **EventSource**: SSE客户端连接
- **轮询机制**: 2秒间隔查询部署状态

### 关键API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/deploy-config/status` | GET | 获取部署状态 |
| `/deploy-config/logs` | GET | 获取部署日志 |
| `/deploy-config/logs/stream` | GET | SSE实时日志流 |
| `/deploy-config/backups` | GET | 列出备份版本 |
| `/deploy-config/deploy` | POST | 开始部署 |
| `/deploy-config/backup` | POST | 创建备份 |
| `/deploy-config/restart` | POST | 重启服务 |
| `/deploy-config/server-logs` | GET | 获取服务器日志 |

## ⚠️ 注意事项

### 部署前检查
1. ✅ 确认网络连接正常
2. ✅ 确认SSH免密登录已配置
3. ✅ 确认远程服务器空间充足
4. ✅ 重要数据已备份

### 部署中注意
1. ⏳ 不要关闭浏览器窗口
2. ⏳ 不要重复点击部署按钮
3. ⏳ 观察实时日志，发现异常及时处理
4. ⏳ 前端构建可能需要30-60秒，请耐心等待

### 部署后验证
1. ✓ 检查部署状态是否为"成功"
2. ✓ 访问前端页面确认功能正常
3. ✓ 查看后端日志确认无报错
4. ✓ 测试关键业务流程

### 故障处理

**问题1: 部署卡住不动**
- 解决: 刷新页面，查看服务器日志，必要时手动重启服务

**问题2: 端口未监听**
- 解决: 查看 backend.log，检查是否有启动错误

**问题3: 前端页面404**
- 解决: 检查 frontend/dist 目录是否存在，Nginx配置是否正确

**问题4: 备份恢复失败**
- 解决: 检查备份文件完整性，磁盘空间是否充足

## 🚀 最佳实践

### 1. 定期备份
- 每次重大更新前创建备份
- 每周至少手动备份一次
- 保留最近10个备份版本

### 2. 分阶段部署
- 先在测试环境验证
- 低峰期进行生产部署
- 准备好回滚方案

### 3. 日志监控
- 部署过程中密切关注实时日志
- 部署完成后检查服务器日志
- 定期清理旧日志文件

### 4. 版本管理
- 使用有意义的备份命名
- 记录每次部署的变更内容
- 建立版本变更记录
