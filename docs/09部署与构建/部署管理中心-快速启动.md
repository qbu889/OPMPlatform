## 🔍 常见问题排查

### 问题1: 页面无法访问

**检查项**:
```bash
# 1. 后端是否启动
ps aux | grep "python app.py"

# 2. 前端是否启动
ps aux | grep "vite"

# 3. 端口是否占用
lsof -i:5004  # 后端
lsof -i:5173  # 前端
```

**解决方案**:
- 确保两个服务都已启动
- 检查端口冲突
- 查看控制台错误信息

### 问题2: SSH连接失败

**症状**: 部署时提示SSH错误或超时

**检查项**:
```bash
# 测试SSH连接
ssh root@8.146.228.47

# 检查SSH密钥
ls -la ~/.ssh/

# 测试SCP
scp test.txt root@8.146.228.47:/tmp/
```

**解决方案**:
- 配置SSH免密登录
- 检查防火墙设置
- 确认远程服务器IP和用户名

### 问题3: 实时日志不更新

**检查项**:
- 打开浏览器开发者工具 (F12)
- 查看 Console 标签是否有错误
- 查看 Network 标签中 `/deploy-config/logs/stream` 请求状态

**解决方案**:
- 刷新页面重新连接SSE
- 检查后端是否正常返回日志流
- 确认CORS配置正确

### 问题4: 备份列表为空

**可能原因**:
- 服务器上还没有创建过备份
- SSH命令执行失败
- 备份目录权限问题

**解决方案**:
```bash
# 手动在服务器上创建备份目录
ssh root@8.146.228.47 "mkdir -p /project/backups"

# 检查目录权限
ssh root@8.146.228.47 "ls -la /project/backups"

# 手动创建一个测试备份
ssh root@8.146.228.47 "cd /project && tar -czf /project/backups/test_backup.tar.gz wordToWord/"
```

## 📊 性能优化建议

### 1. 减少日志数量
如果日志过多影响性能，可以调整保留数量：

```python
# deploy_config_routes.py 中的 add_log 函数
if len(deploy_status['logs']) > 100:  # 改为更小的值，如50
    deploy_status['logs'] = deploy_status['logs'][-50:]
```

### 2. 调整轮询间隔
如果状态更新不够及时，可以减少轮询间隔：

```javascript
// DeployConfig.vue 中的 startStatusPolling
statusPollingInterval = setInterval(() => {
    loadDeployStatus()
}, 1000)  // 从2000改为1000
```

### 3. 使用Redis存储状态
生产环境中，建议使用Redis替代内存存储：

```python
import redis
redis_client = redis.Redis(host='localhost', port=6379, db=0)

# 存储状态
redis_client.set('deploy_status', json.dumps(status))

# 获取状态
status = json.loads(redis_client.get('deploy_status'))
```
