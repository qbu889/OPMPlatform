# 端口配置统一说明

## 📋 概述

为避免端口配置混乱,本项目已将所有后端服务端口统一为 **5004**。

## ✅ 修改内容

### 1. 配置文件

| 文件 | 修改内容 | 说明 |
|------|---------|------|
| `.env` | `PORT=5004` | 本地环境变量配置 |
| `frontend/vite.config.js` | `BACKEND_PORT = 5004` | 前端代理配置 |

### 2. 部署脚本

| 文件 | 修改内容 | 说明 |
|------|---------|------|
| `deploy.sh` | 所有 Nginx proxy_pass 指向 5004 | 主部署脚本 |
| `deploy_with_backup.sh` | 所有 Nginx proxy_pass 指向 5004 | 带备份的部署脚本 |
| `deploy.py` | `LOCAL_PORT = 5004` | Python 部署脚本 |

### 3. 应用代码

| 文件 | 修改内容 | 说明 |
|------|---------|------|
| `app.py` | 生产环境默认端口 5004 | Flask 应用启动配置 |

### 4. 文档

| 文件 | 修改内容 | 说明 |
|------|---------|------|
| `DEPLOY_GUIDE.md` | 所有示例中的端口改为 5004 | 部署指南 |
| `docs/09-部署与构建/MAC_APP_BUILD_GUIDE.md` | 端口示例改为 5004 | Mac App 构建指南 |

## 🎯 为什么选择 5004?

1. **避免 macOS AirTunes 冲突**: macOS 系统占用 5000 端口
2. **符合生产环境默认值**: `app.py` 中生产环境默认使用 5004
3. **远程服务器已在用**: 远程服务器实际运行在 5004 端口
4. **部署脚本已配置**: deploy.sh 中的 Nginx 配置已经是 5004

## 🔧 端口使用情况

| 服务 | 端口 | 说明 |
|------|------|------|
| **后端 Flask** | 5004 | API 服务 |
| **前端 Vite (开发)** | 5200 | 开发服务器 |
| **前端 Nginx (生产)** | 5173 | 生产环境前端 |
| **IOPaint AI** | 8080 | AI 水印清除服务(仅开发) |

## ⚠️ 注意事项

### 本地开发

启动后端时会自动使用 `.env` 中的 `PORT=5004`:

```bash
python app.py
```

如果需要临时使用其他端口:

```bash
export PORT=5001 && python app.py
```

### 远程部署

使用部署脚本自动处理端口配置:

```bash
./deploy.sh
# 或
python deploy.py
```

### 前端代理

前端开发时,Vite 会自动代理到 `localhost:5004`,无需手动配置。

如需覆盖,可设置环境变量:

```bash
BACKEND_PORT=5001 npm run dev
```

## 🐛 常见问题

### Q1: 访问接口返回 502 Bad Gateway?

**原因**: Nginx 配置的端口与后端实际运行端口不一致

**解决**: 
```bash
# 检查后端是否运行在 5004
netstat -tlnp | grep 5004

# 检查 Nginx 配置
grep proxy_pass /www/server/panel/vhost/nginx/sql-formatter-5173.conf

# 如果不一致,更新 Nginx 配置并重启
sed -i 's/127.0.0.1:5002/127.0.0.1:5004/g' /www/server/panel/vhost/nginx/sql-formatter-5173.conf
nginx -t && nginx -s reload
```

### Q2: 前端请求跨域错误?

**原因**: 前端代理配置的后端端口不正确

**解决**: 
检查 `frontend/vite.config.js` 中的 `BACKEND_PORT` 是否为 5004

### Q3: 如何查看当前运行的端口?

```bash
# 查看 Python 进程监听的端口
netstat -tlnp | grep python

# 或使用 lsof
lsof -i:5004
```

## 📝 历史问题

之前存在端口配置混乱的情况:
- 本地 `.env` 使用 5002
- 远程服务器使用 5004
- 部署脚本中混用 5002 和 5004

这导致了多次 502 错误和连接失败。现已统一为 5004,避免此类问题再次发生。

## ✨ 优势

1. **一致性**: 所有环境使用相同端口
2. **简化配置**: 减少环境变量和条件判断
3. **易于维护**: 单一端口,便于监控和调试
4. **避免冲突**: 不与系统常用端口冲突
