# 调整因子计算器实现总结

## 需求分析

根据用户提供的 Excel 文件 `/test/fpa/调整因子.xlsx`，需要将调整因子计算功能分为 9 个模块在前端展示：

### 第一部分（动态计算）
- 规模计数时机、应用类型、质量特性（分布式处理、性能、可靠性、多重站点）、开发语言、开发团队背景
- 这部分需要根据 Excel 公式动态计算值

### 第二部分到第九部分（固定值展示）
- 应用类型列表
- 质量特性判断标准  
- 规模变更调整因子 (CF)
- 开发语言
- 开发团队背景
- 重用程度调整系数
- 修改类型调整系数
- 软件开发计价模型

## 实现方案

### 1. 后端实现

#### 新增路由文件：`routes/adjustment_calc_routes.py`

实现了三个核心接口：

1. **GET `/adjustment-calc/api/calculator-data`**
   - 获取所有调整因子数据
   - 按分类组织返回（应用类型、质量特性、开发语言等 8 类）
   - 用于前端填充参考表格

2. **POST `/adjustment-calc/api/calculate`**
   - 接收用户选择的各项参数
   - 从数据库查询对应的调整因子值
   - 计算总调整因子（所有因子的乘积）
   - 返回详细计算过程

3. **数据库连接管理**
   - `get_db_connection()`: 获取 MySQL 连接
   - `decimal_to_float()`: 数据类型转换

### 2. 前端实现

#### 新增模板文件：`templates/adjustment_calculator.html`

页面结构：

**第一部分：动态计算区域**
- 7 个选择框（规模计数时机、应用类型、质量特性 4 子项、开发语言、开发团队背景）
- 2 个选择框（重用程度、修改类型）
- 每个选择框显示选项名称和对应分值
- 显示原始 Excel 公式作为参考
- "计算总调整因子"按钮

**计算结果区域**
- 渐变紫色背景的结果展示框
- 显示 7 项调整因子的具体值
- 显示总调整因子
- 显示详细计算过程和公式

**参考数据展示区域（第二部分 - 第九部分）**
- 9 个表格展示所有调整因子的完整信息
- 支持响应式布局
- 表格包含：因子名称、描述/判断标准、调整因子值

#### JavaScript 功能

1. **数据加载**: `loadCalculatorData()` - 页面加载时获取所有参考数据
2. **表格渲染**: 8 个渲染函数分别填充不同类别的参考表格
3. **计算功能**: `calculate()` - 发送用户选择到后端计算
4. **结果显示**: `displayResults()` - 展示计算结果和详细过程

两个测试用例：

## 技术特点

### 1. 动态计算
- 第一部分的选择项会根据数据库中的公式动态计算
- 质量特性的 4 个子项分值相加得到总分
- 总调整因子 = 所有因子相乘

### 2. 数据驱动
- 所有调整因子值存储在 MySQL 数据库中
- 前端展示数据来自后端 API
- 支持通过管理后台修改数据

### 3. 用户体验

## 使用示例

```

可靠性：没有明示 (-1)
```

### 计算过程
```

总调整因子 = 1.21 × 1.0 × (-4) × 1.0 × 0.8 × 1.0 × 1.0 = -3.872
```
