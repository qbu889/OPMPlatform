## 📋 功能说明

### 字段映射配置
- **Kafka 字段**：生成的 Kafka 消息中的字段名（如 `STANDARD_ALARM_ID`）
- **ES 字段**：ES 数据中的原始字段名（如 `ALARM_STANDARD_ID`）
- **字段中文解释**：字段的中文说明
- **数据库中文**：数据库中的中文名称

### 工作原理
1. 生成 Kafka 消息时，系统会先查询 `kafka_field_meta` 表
2. 如果找到配置，使用配置的 `es_field` 进行映射
3. 如果没有配置，使用内置的默认映射规则

### 示例
```
配置前：
ES 数据: {"ALARM_STANDARD_ID": "0108-008-012-10-000012"}
Kafka 消息: {"STANDARD_ALARM_ID": ""}  ❌ 未映射

配置后：
添加映射: STANDARD_ALARM_ID -> ALARM_STANDARD_ID
ES 数据: {"ALARM_STANDARD_ID": "0108-008-012-10-000012"}
Kafka 消息: {"STANDARD_ALARM_ID": "0108-008-012-10-000012"}  ✅ 正确映射
```

## 💡 使用场景

### 场景 1：修正错误的字段映射
如果您发现某个字段没有正确映射：
1. 点击"字段映射管理"
2. 搜索该字段
3. 编辑配置，修改 `es_field` 为正确的 ES 字段名
4. 保存后立即生效

### 场景 2：添加新的字段映射
如果需要支持新的字段：
1. 点击"新增映射"
2. 填写 Kafka 字段名和对应的 ES 字段名
3. 保存即可

### 场景 3：临时禁用某个映射
如果某个映射暂时不需要：
1. 找到对应记录
2. 点击"编辑"
3. 将状态改为"禁用"
4. 系统会自动回退到默认映射规则

## ⚠️ 注意事项

1. **唯一性**：每个 Kafka 字段只能有一个映射配置
2. **优先级**：数据库配置 > 内置默认规则
3. **实时生效**：修改后立即生效，无需重启服务
4. **软删除**：删除操作只是禁用，不会真正删除记录

## 🔍 常见问题

### Q: 为什么我的映射没有生效？
A: 请检查以下几点：
1. 确认 `kafka_field` 名称是否正确（区分大小写）
2. 确认 `es_field` 是否在 ES 数据中存在
3. 确认记录状态是否为"启用"

### Q: 如何恢复默认映射？
A: 两种方式：
1. 删除该字段的映射配置（会回退到默认规则）
2. 编辑配置，清空 `es_field` 字段

### Q: 可以批量导入映射配置吗？
A: 目前需要逐个添加，后续可以开发批量导入功能。
