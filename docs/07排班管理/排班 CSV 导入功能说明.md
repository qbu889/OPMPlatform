# 排班表 CSV 导入功能使用说明

## 功能概述
支持从 CSV 文件导入排班数据到系统，导入前会显示预览信息并需要用户确认。

## CSV 文件格式要求

### 必需的列
CSV 文件必须包含以下列:
- **日期**: 支持 `YYYY-MM-DD` 或 `YYYY/MM/DD` 格式 (如：2026-02-24 或 2026/2/24)
- **星期**: 周一、周二等 (用于显示，不影响导入)
- **时段**: 如 8:00～9:00、9:00～12:00 等
- **人员**: 多个人员使用顿号 (、) 分隔

### 可选的列
- **备注**: 请假原因等说明信息

### CSV 文件示例
```csv
日期，星期，时段，人员，备注
2026/2/24，周二，8:00～9:00，陈伟强，
2026/2/24，周二，9:00～12:00，林子旺、陈伟强、吴绍烨，
2026/2/24，周二，13:30～18:00，林子旺、陈伟强、吴绍烨，因请假调整（张三）
2026/2/24，周二，18:00～21:00，陈伟强，
```

## 使用步骤

### 1. 准备 CSV 文件
- 确保 CSV 文件格式正确
- 建议使用 Excel 或其他工具导出为 UTF-8 编码的 CSV 文件
- 如果 Excel 保存，请选择"CSV UTF-8 (逗号分隔) (*.csv)"格式

### 2. 进入排班配置页面
- 访问 `/schedule-config` 页面
- 切换到"排班查询"标签页

### 3. 导入 CSV 文件
1. 点击"导入 CSV"按钮
2. 在弹出的文件选择对话框中选择要导入的 CSV 文件
3. 系统会自动解析文件并显示导入预览:
   - 日期范围
   - 天数统计
   - 记录总数

### 4. 确认导入
- 仔细查看预览信息
- 系统会提示：**此操作将删除指定日期范围内的所有现有排班数据**
- 点击"确定"继续导入，或点击"取消"中止操作

### 5. 查看导入结果
- 导入成功后，系统会自动显示该日期范围的排班数据
- 显示成功消息，包含导入的记录数量

## 注意事项

### ⚠️ 重要提示
1. **数据覆盖**: 导入会先删除指定日期范围内的所有现有排班数据，然后插入新数据
2. **主辅班规则**: 
   - 每个时段的第一位人员会被标记为主班 (is_main=True)
   - 其他人员为辅班 (is_main=False)
3. **多人处理**: 同一时段的多名人员会用顿号分隔，系统会自动拆分为多条记录

### 📝 格式要求
1. **日期格式**: 
   - 支持：`2026-02-24` 或 `2026/2/24`
   - 不支持：`02-24-2026` 或 `24/02/2026`
2. **人员分隔符**: 
   - 使用中文顿号 `、` (如：张三、李四、王五)
   - 也可以使用逗号 `,`
3. **时段格式**: 
   - 使用全角符号 `～` 连接起止时间
   - 如：8:00～9:00、13:30～18:00

### ✅ 最佳实践
1. 导入前先导出当前数据作为备份
2. 在小范围日期测试导入功能 (如 1-2 天)
3. 确认导入无误后再进行大批量数据导入
4. 定期检查导入数据的准确性

## 常见问题

### Q: 导入失败，提示"CSV 文件格式不正确"
A: 检查 CSV 文件是否包含必需的列：日期、星期、时段、人员。注意列名必须完全匹配。

### Q: 导入后人员顺序不对
A: 系统会将 CSV 文件中每个时段的人员列表的第一个人员设为主班，其他为辅班。如需调整顺序，请修改 CSV 文件中的人员排列顺序。

### Q: 导入后发现数据有误，如何恢复？
A: 
1. 如果有备份，可以重新导入备份文件
2. 或者手动在系统中添加/修改排班数据
3. 建议导入前先小范围测试

### Q: 一次最多可以导入多少天的数据？
A: 理论上没有上限，但建议分批导入 (如按月导入),便于检查和验证。

## 技术细节

### 后端 API
- **解析 API**: `POST /schedule-config/api/import-schedule`
  - 接收：CSV 文件 (FormData)
  - 返回：解析后的数据和统计信息
  
- **确认 API**: `POST /schedule-config/api/confirm-import-schedule`
  - 接收：JSON 格式的排班数据
  - 返回：导入结果

### 数据处理流程
1. 读取 CSV 文件 (支持 UTF-8 BOM 编码)
2. 按日期和时段分组数据
3. 解析多个人员 (顿号分隔)
4. 删除目标日期范围的旧数据
5. 批量插入新数据
6. 返回导入统计信息
