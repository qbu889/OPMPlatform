#  Kafka 生成器历史记录功能

> 为 Kafka 消息生成器添加历史记录查看和使用功能

## 🎯 功能简介

在 Kafka 消息生成器的每个字段输入框中新增了【历史】按钮，点击后可以:

- 📋 查看该字段的所有历史生成记录
- 🔍 搜索历史记录 (支持 FP 值、告警名称、地区)
- 📄 分页浏览 (每页 20 条)
- ⚡ 一键使用历史值填充字段

## 🚀 快速开始

### 1. 创建数据库表

```bash
# 方式 1: 使用测试脚本 (推荐，会自动插入测试数据)
python test_kafka_history.py

# 方式 2: 手动创建表
mysql -u root -p knowledge_base < sql/create_kafka_generation_history.sql
```

### 2. 启动应用

```bash
python app.py
```

### 3. 访问页面

打开浏览器访问：`http://127.0.0.1:5001/kafka-generator/`

### 4. 使用功能

在任意字段输入框旁点击【历史】按钮即可使用

## 📊 数据库表结构

### kafka_generation_history 表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键 ID |
| es_source_raw | LONGTEXT | 原始 ES 数据 (JSON) |
| kafka_message | LONGTEXT | 生成的 Kafka 消息 (JSON) |
| created_at | TIMESTAMP | 创建时间 |
| created_by | VARCHAR(100) | 创建者 |
| fp_value | VARCHAR(200) | FP 值 (用于检索) |
| alarm_name | VARCHAR(500) | 告警名称 |
| alarm_level | VARCHAR(50) | 告警级别 |
| region_name | VARCHAR(200) | 地区名称 |

### 索引

- `idx_created_at`: 按时间排序
- `idx_fp_value`: 快速检索 FP 值
- `idx_alarm_name`: 快速检索告警名称

## 🔧 API 接口

### 1. 获取历史记录列表

```
GET /kafka-generator/history
```

**参数:**
- `page`: 页码 (默认 1)
- `per_page`: 每页数量 (默认 20)
- `keyword`: 搜索关键词 (可选)

**响应示例:**
```json
{
  "success": true,
  "data": {
    "list": [
      {
        "id": 1,
        "created_at": "2026-04-01 10:30:00",
        "fp_value": "1713996274_xxx",
        "alarm_name": "设备脱网 (影响 1 条电路)",
        "alarm_level": "2",
        "region_name": "漳州市",
        "kafka_message": {...}
      }
    ],
    "total": 100,
    "page": 1,
    "per_page": 20
  }
}
```

### 2. 获取历史记录详情

```
GET /kafka-generator/history/<id>
```

**响应示例:**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "created_at": "2026-04-01 10:30:00",

    "es_source_raw": {...},
    "kafka_message": {...}
```

## 💡 使用示例

### 查看历史记录

1. 访问 Kafka 生成器页面
2. 找到需要查看的字段 (如：MACHINE_ROOM_INFO)
3. 点击该字段输入框旁的【历史】按钮
4. 查看历史记录列表

### 搜索历史记录

在弹窗的搜索框中输入关键词:
- FP 值：`1713996274`
- 告警名称：`设备脱网`
- 地区：`漳州`

### 使用历史记录

1. 在列表中找到需要的记录
2. 点击该行的 ✓ 按钮
3. 字段值自动填充到输入框
4. 弹窗自动关闭

## 🔍 故障排查

### 问题 1: 点击历史按钮无反应

**解决方案:**
1. 检查浏览器控制台是否有 JavaScript 错误
2. 确认 Bootstrap 库已正确加载
3. 检查网络连接是否正常

### 问题 2: 加载历史记录失败

**解决方案:**
1. 检查 MySQL 数据库是否正常运行
2. 确认 `kafka_generation_history` 表已创建
3. 查看后端日志，确认 API 路由是否正常工作

### 问题 3: 历史记录为空

**解决方案:**
1. 确认已经生成过 Kafka 消息
2. 运行测试脚本插入测试数据：`python test_kafka_history.py`
3. 检查后端日志，确认保存功能是否正常
