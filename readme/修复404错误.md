# 🔧 404 错误修复指南

## 问题描述

```
ERROR:utils.chatbot_core:AI 生成答案失败：AI 服务不可用：404 Client Error: Not Found for url: http://localhost:11434/api/chat
```

## 原因分析

这个错误通常由以下原因引起：

1. **模型配置不匹配** - 代码默认使用 `qwen2.5:7b`，但实际安装的是 `deepseek-r1:7b`
2. **Ollama 服务未启动** - 需要先启动 Ollama 服务
3. **模型未加载** - 模型需要时间加载或从未下载

## 解决方案

### ✅ 方案一：使用配置脚本（推荐）

```bash
cd /Users/linziwang/PycharmProjects/wordToWord
python scripts/setup_ollama_model.py deepseek-r1:7b
```

然后重启应用：
```bash
python app.py
```

### ✅ 方案二：手动设置环境变量

**临时设置（当前终端）：**
```bash
export OLLAMA_MODEL=deepseek-r1:7b
python app.py
```

**永久设置：**
在 `~/.zshrc` 或 `~/.bash_profile` 中添加：
```bash
export OLLAMA_MODEL=deepseek-r1:7b
```

然后重新加载：
```bash
source ~/.zshrc
```

### ✅ 方案三：修改代码默认值

编辑 `utils/ollama_client.py` 第 185 行：

```python
ollama_model = model or os.getenv("OLLAMA_MODEL", "deepseek-r1:7b")  # 改为你的模型
```

## 其他常见问题

### 问题：模型不在列表中

**解决：**
```bash
ollama pull deepseek-r1:7b
```

### 问题：Ollama 服务未运行

**解决：**
```bash
ollama serve
```

在新终端窗口运行此命令，保持后台运行。

### 问题：仍然报 404 错误

**解决：** 代码已自动添加备用方案，会切换到 `/api/generate` 端点。如果还有问题，查看日志：

```bash
# 查看详细日志
tail -f logs/app.log  # 如果有日志文件
```
