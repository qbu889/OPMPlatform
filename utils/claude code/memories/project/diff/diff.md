# JSON 数据对比工具需求文档

##  项目概述

### 背景
在实际开发中，经常需要对比两个JSON数据文件的差异。但由于JSON对象的键值对顺序可能不同，直接使用文本diff工具无法准确识别内容差异。需要一个智能的JSON对比工具，能够忽略字段顺序，准确对比字段值的差异。

### 目标
开发一个可视化JSON对比工具，支持：
- 智能识别相同字段（忽略顺序）
- 高亮显示差异（绿色=相同，红色=不同）
- 支持市面上主流diff工具的常见功能

- ✅ 支持从URL加载JSON数据

**优先级：** P0（核心）

### 2. 智能字段匹配
**功能描述：** 忽略JSON对象中字段的顺序，准确匹配相同字段

**具体要求：**
- ✅ 递归对比JSON对象的所有层级
- ✅ 字段按字母顺序或原始顺序对齐显示
- ✅ 自动识别新增字段（仅存在于右侧）
- ✅ 自动识别删除字段（仅存在于左侧）
- ✅ 支持数组元素的顺序无关对比（可选）

**示例：**
```json
// 左侧JSON
{
  "name": "张三",
  "age": 25,
  "city": "北京"
}

// 右侧JSON（字段顺序不同）
{
  "city": "北京",
  "name": "张三",
  "age": 26  // 仅这个字段不同
}
```
**对比结果：** `name`和`city`标记为绿色（相同），`age`标记为红色（不同）

**优先级：** P0（核心）

### 3. 差异高亮显示
**功能描述：** 使用颜色标记相同和不同的字段值

**具体要求：**
- 🟢 **绿色背景**：字段值完全相同
- 🔴 **红色背景**：字段值不同或字段缺失
-  **蓝色背景**：字段仅存在于一侧（新增/删除）
-  **灰色背景**：字段类型为对象或数组（可展开）

**显示样式：**
```diff
# 左侧                              | 右侧
{                                   | {
  "name": "张三"      [绿色]        |   "name": "张三"      [绿色]
  "age": 25           [红色]        |   "age": 26           [红色]
  "city": "北京"      [绿色]        |   "city": "北京"      [绿色]
                                    |   "phone": "138..."   [蓝色-新增]
  "email": "..."     [蓝色-删除]    |
}                                   | }
```

**优先级：** P0（核心）

### 5. 高级对比选项

#### 5.1 对比模式配置
**功能描述：** 支持不同的对比策略

**具体要求：**
- ✅ **严格模式**：类型和值都必须完全相同

- ✅ **数组对比模式**：

  - 顺序无关（集合对比）
  - 按主键匹配（指定某个字段作为主键）

**具体要求：**
- ✅ 支持JSONPath语法（如 `$.users[0].name`）
- ✅ 输入路径自动跳转到对应字段
- ✅ 点击字段显示完整路径

**优先级：** P3

### 核心算法
```javascript
// 伪代码示例
function compareJSON(left, right, options = {}) {
  const result = {
    same: [],      // 相同字段
    different: [], // 不同字段
    added: [],     // 新增字段
    removed: []    // 删除字段
  }
  
  // 递归对比
  function deepCompare(obj1, obj2, path = '') {
    const allKeys = new Set([...Object.keys(obj1), ...Object.keys(obj2)])
    
    for (const key of allKeys) {
      const currentPath = path ? `${path}.${key}` : key
      
      if (!(key in obj1)) {
        result.added.push({ path: currentPath, value: obj2[key] })
      } else if (!(key in obj2)) {
        result.removed.push({ path: currentPath, value: obj1[key] })
      } else if (isEqual(obj1[key], obj2[key])) {
        result.same.push({ path: currentPath, value: obj1[key] })
      } else {
        result.different.push({
          path: currentPath,
          left: obj1[key],
          right: obj2[key]
        })
        // 如果是对象，递归对比
        if (isObject(obj1[key]) && isObject(obj2[key])) {
          deepCompare(obj1[key], obj2[key], currentPath)
        }
      }
    }
  }
  
  deepCompare(left, right)
  return result
}
```

- **deep-diff**：深度对象对比
- **fast-json-patch**：JSON Patch标准实现

## 📊 使用场景示例

### 场景1：API响应数据对比
```json
// 修改前的API响应
{
  "status": "success",
  "data": {
    "user": {
      "id": 123,
      "name": "张三",
      "role": "admin"
    }
  }
}

// 修改后的API响应
{
  "status": "success",
  "data": {
    "user": {
      "id": 123,
      "name": "张三",
      "role": "super_admin",  // 权限升级
      "email": "zhangsan@example.com"  // 新增字段
    }
  }
}
```

### 场景2：配置文件版本对比
```json
// v1.0 配置
{
  "database": {
    "host": "localhost",
    "port": 3306,
    "pool_size": 10
  }
}

// v2.0 配置
{
  "database": {
    "host": "db.example.com",  // 改为远程数据库
    "port": 3306,
    "pool_size": 50,  // 连接池扩大
    "ssl": true  // 新增SSL配置
  }
}
```

## 📝 附录

### 术语说明
- **JSON Path**：JSON数据的路径表达式，如 `$.users[0].name`
- **递归对比**：对嵌套的JSON对象进行逐层对比
- **差异（Diff）**：两个JSON数据之间的不同之处
- **补丁（Patch）**：将一个JSON转换为另一个JSON的操作集合

### 相关标准
- **RFC 6901**：JSON Pointer标准
- **RFC 6902**：JSON Patch标准
- **RFC 7396**：JSON Merge Patch标准
