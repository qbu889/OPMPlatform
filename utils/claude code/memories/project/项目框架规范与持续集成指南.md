## 1. 项目概述

### 1.1 项目简介

诺基亚 OPM 综合业务系统是一个企业级综合业务平台，提供以下核心功能：

- **文档转换**: Word/Excel/Markdown 互转、COSMIC 转换、ES 数据导出
- **Kafka 消息生成**: 自定义字段、历史缓存、推送消息生成
- **FPA 功能点估算**: SQL 转换、事件分类、规则管理
- **智能客服**: AI 对话、知识库管理、文档问答
- **排班管理**: 定时推送、钉钉集成
- **在线表格**: 协同编辑、数据导出
- **图片水印清除**: AI 智能去水印（IOPaint LaMa 模型）

### 1.2 架构模式

```
┌─────────────────────────────────────────────┐
│              前端 (Vue 3 SPA)               │
│         Vite + Vue Router + Pinia           │
└─────────────────────────────────────────────┘
                    ↓ API 请求
┌─────────────────────────────────────────────┐
│        后端 (Flask REST API)                │
│     Flask + Blueprint + MySQL + Redis       │
└─────────────────────────────────────────────┘
                    ↓ 外部服务
┌─────────────────────────────────────────────┐
│    AI 服务        │   MySQL     │   Kafka   │
│  (OMLX/Ollama)   │  数据库     │  消息队列 │
└─────────────────────────────────────────────┘
```

### 3.1 路由模块规范

每个功能模块必须遵循以下结构：

```
routes/
└── 模块名/
    ├── __init__.py            # 导出 Blueprint
    └── 模块名_routes.py       # 路由定义
```

**Blueprint 注册示例**：

```python
# routes/kafka/kafka_generator_routes.py
from flask import Blueprint, jsonify, request

kafka_generator_bp = Blueprint(
    'kafka_generator_bp',
    __name__,
    url_prefix='/kafka-generator'
)

@kafka_generator_bp.route('/generate', methods=['POST'])
def generate_kafka_message():
    """生成 Kafka 消息"""
    pass
```

**主应用注册**：

```python
# app.py
from routes.kafka.kafka_generator_routes import kafka_generator_bp

blueprints = [
    kafka_generator_bp,
    # ... 其他模块
]

for blueprint in blueprints:
    app.register_blueprint(blueprint)
```

采用 Git Flow 工作流：

```
master (生产)
  ↑
develop (开发)
  ↑
feature/xxx (功能分支)
  ↑
bugfix/xxx (bug 修复)
  ↑

```

```bash

# Bug 修复
```

#### 5.1.2 Commit Message 规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型 (type)**:
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试用例
- `chore`: 构建/工具变动

**示例**:

```bash
feat(kafka): 添加自定义字段持久化功能

- 新增字段缓存到 MySQL
- 支持字段置顶排序
- 添加历史记录弹窗

Closes #123
```

### 7.1 GitHub Actions 配置

#### 7.1.1 完整 CI/CD 流程

```yaml
# .github/workflows/ci-cd.yml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, master, develop ]
  pull_request:
    branches: [ main, master, develop ]

  # 代码质量检查
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.13'
      - name: Install linting tools
        run: |

      - name: Run flake8
        run: flake8 routes/ utils/ models/ --max-line-length=120
      - name: Run black check
        run: black --check routes/ utils/ models/
      - name: Run isort check
        run: isort --check-only routes/ utils/ models/

  # 前端构建检查
  frontend-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json
```

## 9. 常见问题

### 9.1 开发环境问题

#### Q: 前端代理 502 错误

**原因**: 后端服务未启动或端口不匹配

**解决**:
```bash
# 检查后端端口
lsof -i :5001

# 检查前端代理配置
cat frontend/vite.config.js

# 确保后端和前端端口匹配
# vite.config.js 中的 BACKEND_PORT 应该与实际后端端口一致
```

#### Q: 数据库连接失败

**原因**: MySQL 服务未启动或配置错误

**解决**:
```bash
# 检查 MySQL 服务
brew services list | grep mysql

# 启动 MySQL
brew services start mysql

# 测试连接
mysql -u root -p -h localhost
```

#### Q: 导入模块找不到

**原因**: Python 路径问题

**解决**:
```bash
# 确保项目根目录在 Python 路径中
export PYTHONPATH=/Users/linziwang/PycharmProjects/wordToWord:$PYTHONPATH

# 或者在代码中添加
import sys
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))
```

### 9.2 测试问题

#### Q: 测试找不到 fixtures

**解决**: 确保 `conftest.py` 在测试目录中

```python
# test/conftest.py
import pytest
from app import create_app

@pytest.fixture
def app():
    app = create_app('testing')
    yield app

@pytest.fixture
def client(app):
    return app.test_client()
```

#### Q: 测试覆盖率低

**解决**:
1. 为新功能编写测试用例
2. 使用 `--cov-report=term-missing` 查看未覆盖的代码行
3. 优先测试核心业务逻辑

### 9.3 部署问题

#### Q: 生产环境静态文件 404

**解决**: 确保前端构建产物已复制到后端静态目录

```bash
# 构建前端
cd frontend && npm run build

# 复制产物
cp -r dist/* ../static/
```

#### Q: 数据库迁移

```bash
# 使用 Alembic 管理数据库迁移
pip install alembic

# 初始化迁移
alembic init alembic

# 创建迁移脚本
alembic revision --autogenerate -m "add new table"

# 应用迁移
alembic upgrade head
```
